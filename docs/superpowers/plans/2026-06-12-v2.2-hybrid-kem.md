# ShortyQ v2.2 Hybrid KEM Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Hybrid X25519 + ML-KEM-768 key exchange (OpenSSH-style posture); hybrid keys by default, all legacy keys/payloads keep working.

**Architecture:** Keys are length-tagged concatenations (hybrid pk 1216 = mlkem 1184 ‖ x25519 32; hybrid sk 2432 = 2400 ‖ 32). Hybrid payloads carry `x25519Ciphertext` (ephemeral pk). Base AES key = `sha256("shortyq-hybrid-v1" ‖ ssM ‖ ssX ‖ ctX ‖ pkX)`; password derivation composes on top unchanged.

**Tech Stack:** Existing stack + `@noble/curves@^1.9.0` (`x25519` from `@noble/curves/ed25519`), already installed and CJS-verified.

**Reference — spec:** `docs/superpowers/specs/2026-06-12-v2.2-hybrid-kem-design.md`
**Branch:** `v2.2-hybrid` (created). Plain commit messages, no attribution.

export function generateKeyPair(): KeyPair {
  const { publicKey, secretKey } = ml_kem768.keygen();
  const xSecret = x25519.utils.randomPrivateKey();
  const xPublic = x25519.getPublicKey(xSecret);
  return {
    publicKey: toBase64(concatBytes(publicKey, xPublic)),
    secretKey: toBase64(concatBytes(secretKey, xSecret)),
    codeKey: toBase64(new Uint8Array(randomBytes(CODE_KEY_BYTES))),

}

```typescript
function concatBytes(...arrays: Uint8Array[]): Uint8Array {
  const out = new Uint8Array(arrays.reduce((n, a) => n + a.length, 0));
  let offset = 0;
  for (const a of arrays) {
    out.set(a, offset);
    offset += a.length;
  }
  return out;
}
```
